## chap7 要求仕様書の書き方
* よい要求仕様が満たすべき特性として以下の項目がある．
* 妥当である．
* あいまいでない．
* 完全である．
* 一貫性がある．
* 重要度と安定性のランク付けがされている．
* 検証可能である．
* 変更可能である．
* 追跡可能である．

要求仕様の完全性
1. 機能，性能，設計上の制約，属性，外部インタフェースに関する重要な要求．
2. あらゆる実現可能な状況において，入力データにソフトウェアがどのように応答するかの定義．特に，正当な入力値と不当な入力値の両方に対する応答を記述することが重要である．
3. 要求仕様中の図や表に対するラベル付と参照，および，用語と単位の定義．

* なぜ未定事項となったのか，未定事項の原因となった要因
* 未定事項を解消するために何をすればよいか
* 誰が解消すべきか
* いつまでに解消すべきか
